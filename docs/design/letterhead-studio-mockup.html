THE MODEL — two axes, three tiers

STYLE (how it looks: logo/color/type) is a pure presentation layer over data the core already returns — sac, app-side, no LLM.  •  STRUCTURE (what’s in it: sections/fields/new doc types) keeps a named schema, so the fill stays a solved mapping; custom fields thread into the extraction prompt (light core).  •  UPLOAD (bring any document) is the hard one — the LLM must comprehend an unknown document, so it’s gated as Premium (token cost) and made reliable by infer-schema → confirm-once, never freeform-every-time.

TIERS. Free: presets + basic branding + “Prepared with Jefe” footer.   Pro: remove footer, custom fields, more fonts.   Premium: upload your own document (comprehension tokens).  •  THE ONE RULE: the LLM only ever fills a named schema — authored (Structure) or inferred-then-confirmed (Upload). It never guesses a raw document live, because that’s someone’s bill.
